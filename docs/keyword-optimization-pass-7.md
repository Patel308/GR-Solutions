## Implementation summary

- **40 service diagnostics:** two additional symptom or decision topics for each of five TV service types across four cities.

- **20 city-wide intents:** five additional practical repair topics across the four generic city pages.
- All topics are rendered as visible supporting guidance through the existing shared service page component.
- Existing metadata titles, descriptions, H1s, canonical URLs, structured-data service identities and sitemap URLs remain unchanged.
- No fake authorization, guaranteed same-day visit, fixed-price promise, rating, review or certification claim was introduced.
- No keyword received a separate page where the existing canonical page already satisfies the same search intent.

## Route decision

No new route is required for this pass. The site keeps its existing architecture:

- service symptoms map to the matching service-city page;
- brand feature and fault terms map to the matching brand-city page;
- broad booking and diagnostic terms map to the matching generic city repair page.

This brings the documented cumulative semantic coverage to **500 targets**, while preserving one canonical page per meaningful search intent.
